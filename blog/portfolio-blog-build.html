个人博客从 0 到 1：我如何搭建自己的 AI 学习型作品集

首次发布: 2026-05-20 03:54:11
修改历史记录
2026-05-20 03:54:11 首次发布: 博客开发复盘日志正式上线
· 8 min read · 15 views · Beginner · Growing
Portfolio Blog HTML CSS JavaScript Developer Hub

作为一个本科阶段人工智能方向的学习者，拥有一个精美且内容充实的个人作品集和学习日志平台是我一直以来的想法。在日常的学习、科研实验和考研备考中，沉淀下来的笔记和反思如果仅存放在本地的 Markdown 编辑器里，未免有些可惜。因此，我决定自己动手，搭建这套专属于我的 AI 学习型作品集门户。

“最好的学习方式就是输出。把学过的技术以博客形式讲给别人听，不仅能检验自己的理解，更是建立个人技术影响力的第一步。”

一、为什么我要做这个个人博客

市面上有很多成熟的静态博客生成器，如 Hexo、Hugo，或者高度封装的博客平台。但对我而言，我不仅想要一个单纯的文字发布渠道，更需要一个展示个人科研成果、项目经历、备考进度的多维度作品集门户。从零手写前端代码能够让我对每一像素的控制、每一种主题切换和底层路由机制都有绝对的掌控权。这本身就是一次非常棒的 Web 前端交互开发实践。

二、从 Bento Grid 首页开始

我的设计灵感来源于现代化的 Bento Grid（便当盒）卡片布局。我通过 Vanilla CSS Grid 将主页划分为多个功能各异的玻璃态微动效卡片。每一个卡片代表作品集的一部分，包括个人简介、GitHub 动态、专业技能、经历时间线以及博客动态。当鼠标滑过这些卡片时，由于设置了极富空间感的高斯拉伸模糊阴影与微移交互效果，页面呈现出一种呼吸式的灵动感觉。

三、把博客从单篇文章升级成 Blog Hub

原本的设计非常简单，首页的“博客动态”仅直接链接到单篇静态 HTML 文章。这在博文较少时还算适用，但随着我计划输出更多的学习日志，必须构建一个完整的博客汇聚中心 (Blog Hub)。

为此，我单独开辟了 blog/index.html 作为博客首页。它负责从统一的数据结构中读取文章元数据，并动态生成交互卡片。为了避免频繁地通过手动添加 HTML 节点的低效劳动，我将数据部分完全抽离。

四、为什么要设计 6 个固定分类

为了让知识体系更加系统化，我在 Blog Hub 中将所有的内容归纳为以下 6 个固定的分类：

  • AI Learning：记录深度学习、PyTorch 科研实验与大模型微调笔记。
  • Python Notes：整理 Python 基础、数据采集与自动化处理脚本。
  • CS Fundamentals：复习 408 考研中的数据结构、操作系统与计网计组。
  • Project Logs：记录个人开源项目的迭代、架构演进与踩坑记录。
  • Study Roadmap：分享考研备战时间表、学习路线图与方法论。
  • Life & Reflection：在喧嚣中保留一块自留地，书写阶段总结与成长焦虑。

五、纯 HTML + CSS + JS 的好处

在这个项目中，我坚持使用纯粹的 HTML5、CSS3 与原生 JavaScript，不引入任何构建工具（如 Webpack、Vite）或现代庞大的前端框架（如 React、Vue）。好处是显而易见的：

  1. 极速的加载性能：没有任何框架包的体积负担，直达浏览器渲染引擎。
  2. 100% 的可移植性：整个项目只要有浏览器就能运行，无需本地配置 Node 环境。
  3. 易于部署：可以直接放在 GitHub Pages、GitHub Action 或者本地任何静态托管服务中。

六、本次项目中遇到的路径迁移问题

为了保证作品集在本地、服务器或未来迁移至其他盘符下运行时不受硬编码绝对路径的影响，我把所有的资源引入都改为了严格的相对路径。这要求我在编写代码时对当前的页面层级有清晰的逻辑认知。例如，在子页面加载博客数据与通用交互逻辑时，我采用了以下的引入方式：

HTML - 相对路径载入示例 
<!-- 引入全局外部样式表 -->
<link rel="stylesheet" href="../styles.css">

<!-- 先载入博客静态结构化数据 -->
<script src="./blog-data.js"></script>

<!-- 再载入具有 Null Checks 的交互及渲染脚本 -->
<script src="./blog.js"></script>

七、我学到的东西

通过解决各种样式渲染冲突和跨页面交互需求，我极大地提高了自己的前端功底：

  • 多页面主题同步：通过 localStorage 完美解决了跳转页面时屏幕闪烁以及主题断档的问题。
  • 安全健壮性：理解了在公用 JavaScript 脚本中做 null check 的必要性。由于详情页没有 Blog Hub 的搜索框和过滤芯片，若不做判定直接获取 DOM，脚本就会报错断流，使底部的返回与代码复制功能直接罢工。

八、下一步计划

未来，我不仅会继续在此发布 408 考研的知识卡片，还会结合我目前正在助研的 AI 实验室项目，产出更多有关大模型微调、RAG 向量检索等干货技术文章，把它真正打造成我的 AI 学习知识库与个人作品集门户。
